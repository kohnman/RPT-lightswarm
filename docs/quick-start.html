Quick Start Guide

Get the River Park Towers lighting system running in under 5 minutes

LightSwarm

USB-connected controller (optional for testing)

No hardware? No problem! The system runs in Simulation Mode automatically when no hardware is detected.

1

Start the System

Open Terminal and run these commands:

# Navigate to middleware folder and install dependencies
cd middleware
npm install # First time only
npm start

You should see:

RiverPark LightSwarm Middleware v1.0.0
==================================================
Middleware initialized successfully!
API Documentation: http://localhost:3000/api/docs
Admin Dashboard: http://localhost:3000/
==================================================
Tip: If you see "Simulation mode enabled", that's normal when no LightSwarm hardware is connected.
2

Open the Dashboard

Open your web browser and go to:

http://localhost:3000

Green indicator

System is ready

Simulation

"Simulation" badge

Running without hardware (normal for testing)

3

Run Your First Test

Quick Test (No Setup Required)

  1. 1 Click the Test Console tab
  2. 2 In the "Address" field, enter: 1
  3. 3 Select command: Set RGB
  4. 4 Set colors: Red=255, Green=0, Blue=0
  5. 5 Click Send Command
Expected result: The packet hex code appears below the button. In simulation mode, the command is logged but no physical light changes.

Broadcast Test (All Lights)

  1. 1 In Test Console, find Broadcast Commands
  2. 2 Click All ON
  3. 3 Click All OFF
4

Import Apartment Data (Optional)

If you have the apartment Excel file:

  1. 1 Click the Import tab
  2. 2 Drag and drop the Excel file (.xlsx)
  3. 3 Click Preview to verify the data
  4. 4 Click Import to load apartments

"Cannot connect to localhost:3000"

lsof -i :3000 # Find what's using the port
kill -9 <PID> # Kill that process
npm start # Try again

"Serial port error"

This is normal if no LightSwarm hardware is connected. The system automatically switches to simulation mode.

Dashboard shows blue/blank screen

  • ‚Ä¢ Open browser developer tools (F12)
  • ‚Ä¢ Check Console tab for JavaScript errors
  • ‚Ä¢ Try a hard refresh: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
